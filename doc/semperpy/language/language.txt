:mod:`The Language class` --- Language validation
=================================================

.. module:: language.language

.. versionadded:: 0.1
.. moduleauthor:: Claude Gibert <claude.gibert@synopticview.co.uk>

.. class:: LanguageError(Exception)

    Defines an exception class to catch at higher levels if the error has to do with
    language checking. Other exceptions can be raised in this module

.. class:: Language(object)

    The **Language** class is not meant to instanciated, it implements class methods
    to read language definition modules and files and then apply these to a dictionary.

    The **Language** class offers one single public method. This method uses three
    pieces of information to "do the job":

    * a dictionary provided by the user which needs to be analysed and matched against
      rules (e.g. default values, required keywords, type conversions etc...)
    * a language definition defining all these rules
    * language definition rules which define rules for the language definition and default
      behaviour when information is not provided in the language (see the :class:`Language.LanguageDefinition.LanguageDefinition` class).

    This may sound complicated but the user should never need to look at the third component: 
    the language definition rules. The second component is only known to users 
    who design a language.

Public methods
--------------

    .. classmethod:: resolveDirective(directive,directive_name,directive_reader):

    	:param directive: the data to be checked against the language rules
    	:type directive: a dictionary or any object behaving like a dictionary
    	:param directive_name: a name which enables the directive reader to provide the language rules.	
    	:type directive_name: string
    	:param directive_reader: an object able to return a dictionary compliant with what is defined in class:: `LanguageDefinition`. It receives the string *directive_name*. This means that the language definition can be stored in any format, or even be a python module.
    	:type directive_reader: any callable object which returns the language definition from a single string parameter

    	The class method :meth:`resolveDirective` raises a :class:`Language.Language.LanguageError` exception if any of the steps goes wrong. However, it continues as far as possible and only raises the exception with a list of all errors encountered to facilitate debugging. It follows the following steps:

    	* retrieve the language definition through the directive_reader. This is a recursive 
    	  process, if other language definitions are inherited, they are merged together,
    	* check the correctness of the language definition(s) in syntax and grammar,
    	* *default language keywords* which were not provided in *directive* are inserted,
    	* *aliases* are substituted for the *legal keyword*
    	* *default values* which were not provided are inserted
    	* the presence of required keywords is checked
    	* keywords which are allowed only one value are checked, keywords with more than 
    	  one value are converted to lists,
    	* for each keyword, if a type (or class) is specified, the value is converted,
    	* values including environment variables are substituted with the value of the variable(s)
    	* string values are converted to lower case, unless specified otherwise
    	* at this stage an exception can be raised if at least one error has been detected in 
    	  the previous stage.
    	* if no error occured, then one last step is necessary: 
    	  to run *validators* on values to make sure everything is ok, 
    	  values can be derived etc...,
    	* if not error is encountered, the directive cleaned up and checked is returned.

    	The errors generated by this method are:

    		1. unknown keywords (not defined in the language),
    		2. keywords with more one than one value which should only have one,
    		3. errors while trying to convert values to specified types,
    		4. errors while trying to substitute values with environment variables 
    		   (environment variable missing),
    		5. validation of the value(s) of a keyword failed.

    	:note: At this stage it seems more interesting to gather as many errors as possible in one single :class:`language.language.LanguageError` exception rather than generating specialised exceptions for each type of error.

    	:warning: The *directive* sent as the first parameter is the one which is returned, it is modified by this method. If you want to preserve it, you will need to duplicate it before calling this method. This is done deliberateley to avoid the systematic duplication of the parameter if it is not necessary.

        :note: The parameter **directive** is expected to have two attributes: default\_ and keywords\_. They should both be instances of the **set** class. On return they contain respectively the list of keywords which received a default value and the list of keywords valid in this directive.

Private methods
---------------

    .. classmethod:: validate_language(directive_name,directive_reader):
    	
    	:param directive_name: name of the language definition (probably the name of the directive)
    	:type directive_name: string
    	:param directive_reader: an object able to return a dictionary compliant with what is defined in class:: `LanguageDefinition`. It receives the string *directive_name*. This means that the language definition can be stored in any format, or even be a python module.
    	:type directive_reader: any callable object which returns the language definition from a single string parameter

    	This method gets the language definition for a directive through the directive_reader,
    	deals recursively with inherited language definitions if specified. It checks the validity
    	of the language definitions and builds a final language definition from inherited
    	definitions.

    	Returns: the final language definition, the definition of how the language should 
    	describe the directive and the definition of how the keywords should be defined.
    	
    .. classmethod:: inherit_from(parent,child):
    
    	:param parent: the dictionary to inherit from
    	:type parent: dictionary
    	:param child: the dictionary to inherit into
    	:type child: dictionary

    	Inserts into child any entry of parent which is not in child. From entries which are
    	in child and in parent, inherit all the entries in the entry from parent into
    	the child.

    .. classmethod:: specialize_from(parent,child):

    	:param parent: the dictionary to override values from
    	:type parent: dictionary
    	:param child: the dictionary to ovveride into
    	:type child: dictionary

    	Inserts into child any entry of parent which is not in child. Replaces in
    	child any entry which is both in the parent and the child.

    .. classmethod:: validate_language_part(semantic,language):

    	:param semantic: the definition of how the language should be defined
    	:type semantic: dictionary
    	:param language: the language to check
    	:type language: dictionary

    	Checks that the language definition contains all the required keywords and
    	ensures that the different entries are of the correct type.

    	Returns the language after modification.

    .. classmethod:: resolveSpecialKeywords(definition,directive,language):

        :param definition: A dictionary written in the language definition under the keyword **specialize_from**.
        :type definition: dictionary
        :param directive: the directive given by the user.
        :type directive: dictionary
        :param language: the language being used to verify the directive
        :type language: dictionary

        This method tries to extract keywords from the different expressions in
        **definition** which are defined in the language. It then applies the 
        language definition to those keywords in order to make sure that, before
        evaluating the expressions, the keywords exist (possible default
        values) and are of the expected type.

        In order to apply the language to a reduced set of keyword, this method
        creates a reduced directive and builds in memory a reduced language
        definition.

    .. classmethod:: find_specials(definition,directive):

        :param definition: A dictionary written in the language definition under the keyword **specialize_from**.
        :type definition: dictionary
        :param directive: the directive given by the user.
        :type directive: dictionary

        This methods evaluates expressions defined under the **specialize_from** in the
        language definition. The syntax is as follows::
            
            ...
            specialize_from = {
                "value1 == 'hello' and value2 == 'world'": "hello",
                "value3 < 12": "not_twelve"
            }
            ...

        Where the keys are boolean expressions which can name any key in the user's
        directive and the values are the name of directive definitions. The contents
        of the directive definition is loaded and its keys replace the current
        keys if they exist or they are added. If the evaluation of the expression
        if False, nothing is done.

        This is useful to define the semantics of directive depending on values of
        some of its values.

    .. classmethod evaluate_condition(condition,directive):

        Evaluate an expression and returns a Boolean value.

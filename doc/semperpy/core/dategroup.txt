:mod:`dategroup` --- Date grouping tools
========================================

.. versionadded:: 0.1
.. moduleauthor:: Claude Gibert <claude.gibert@synopticview.co.uk>

The DateGroup class
--------------------

.. class:: DateGroup(object)

    The **DateGroup** class is an interface to different actors which are able
    to group dates accoring to different criteria. Actors are also implemented in 
    this module::

        from semperpy.core.date import Dates
        from semperpy.core.dategroup import DateGroup

        def displayData(groups):
            for group in groups.iterator():
                r = []
                for date,value in group:
                    r.append(str(date) + ' ' + str(value))
                print ', '.join(r)

        d = Dates(2011010100,2011010112) + Dates(2011010200,2011010212)
        g = DateGroup.createGroup('hour',d)
        displayData(g)

    The data is now grouped by chunks of 3 hours. The data can be grouped:

        * per hour (hour)
        * hourly (hourly,N)
        * per day (day)
        * daily (daily,N)
        * per month (month)
        * monthly (monthly,N)
        * per year (year)
        * yearly (yearly,N)

    This grouping is designed to help with iterating over dates when some meaningful organisation
    is needed::

        d = Dates(2011010100,2011010112) + Dates(2011010200,2011010212)
        for i in range(len(d)):
            data[d[i]] = i + 1
        g = DateGroup.createGroup('hourly',data,3)
        displayData(g)
        print

        displayData(g)

    An example of application of this is when staging data from an archive. Files are stored in an
    archive, each file corresponds to data for a particular date-time. You suspect that all files
    from the same day are probably on the same tape and you know that it is safe to stage a whole
    day. You organise all the files involved in a dictionary which keys are the date and the value
    is the filename. You group the dictionary daily and write the loop::

        for group in groups.iterator():
            files = group.values()
            stage(files)
            for date,filename in group:
                # do whatever 

    This way if you find that grouping the data 6 hourly or even 12 hourly, there is no need to
    change anything in the algorithm.

    .. classmethod:: createGroup(name,container = None, count = 1)

        :param name: name of the grouping method
        :type name: string
        :param container: a list of dates or a { date: data }
        :type container: list of dict
        :param count: the number of basic group units
        :type count: int

        This method returns a DateGroup instance. The container can either be a list of date or a 
        dictionary which keys are dates and values can be or any arbitrary type.

    .. method:: iterator()

        Returns an iterators on the groups which attempts to iterate chronologically. For each
        iteration, an iterator is returned. That iterator returns tuples (date,value) for each 
        couple present in a group by chronological order::

            d = Dates(2011010100,2011010112) + Dates(2011010200,2011010212)
            g = DateGroup.createGroup('hourlyGroup',d,6)
            for group in g.iterator():
                for date, value in group:
                    # do something
                # do something you want to do for each group of 6 hours.

        The iterator returned for each group has also two methods which are dictionary-like:

            * keys(): returns a sorted list all keys
            * values(): returns a list of all values in the order of sorted keys.

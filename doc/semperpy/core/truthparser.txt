:mod:`truthparser` --- Parsing Boolean expressions
====================================================

.. module:: core.truthparser
   :synopsis: A parser and computation engine for boolean expressions

.. versionadded:: 0.1
.. moduleauthor:: Claude Gibert <claude.gibert@synopticview.co.uk>

The TruthParser class
----------------------

.. class:: TruthParser(dict)

    The TruthParser analyses boolean expressions in the form: "**a == 1 and b < 3 or c == 0**".
    If then builds a tree from the reverse polish notation of the expresion, loading each
    node and leaf with user-defined object instances. It is then able to "execute" the 
    tree to let the user's objects do their job.

    The TruthParser uses pyparser Python module (http://pyparsing.wikispaces.com/) and handles
    comparison operator and logical operators: **==, !=, <, <=, >, >=** together with **and** and **or**.
    It is quite simple and crude and could be improved greatly particularly for handling
    user's syntax errors, but it does the job it was required to do. It can be improved
    later.

    Some sort of abstraction has been put into developping the parser, it could be used
    for most applications which require truth parsing. The user of this class should implement
    a class with the following methods (what is the token parsed in the expression):

        * createLogical(self,what): returns an object instance to deal with **x and y** or **x or y**
        * createComparison(self,what) returns an object instance to deal with operator **==, !=, <, <=, >, >=**
        * createVariable(self,what) returns an object instance to deal with variables defines in the expression, basically any string which is not a number or an operator
        * createValue(self,what) returns an object instance to deal with hardcoded floating point values which are part of the expression.

    The TruthParser class provides default behaviour for nodes holding a logical (and, or),
    a comparison (<,>,== etc...) and a value (literal floating point value). Nothing can be supplied
    by default for Variable.

    Once the tree representing the expression is built, it is executed to produce a result. The 
    following object methods are called during the process:

        * Object created by createLogical: __call__(self,a,b) returns the result of the logical operation between a and b (or whatever they contain)
        * Object created by createComparison: __call__(self,a,b) returns the result of the comparison of a and b
        * Object created by createVariable: __call__(self) returns the value of the variable in the expression
        * Object created by createValue: __call__(self) returns the value.

    .. method:: parse(expression,creator)

    	:param expression: the text expression to be parsed.
        :type expression: string
    	:param creator: called with one parameter, the token found, should return an object instance to be stored in a node.
        :type creator: an object instance with the necessary methods see introduction

        Parses the text, creates the expression tree and executes it calling the objects provided
        by the creator (please see introduction).

    .. method:: build(x,creator)

        Builds the expression tree, *called internally*.

    .. method:: execute(tree)

        Executes the tree by calling the object instances, *called internally*.
